# screenwriter 审计引擎

本模块保留 screenwriter 的剧作结构方法：麦基、坎贝尔、亚里士多德、场景价值转变、因果性审计、精准修改、一个版本原则。

## 适用场景

- 用户已有梗概、分场、人物设定或剧本片段。
- 需要判断故事是否站得住。
- 需要修改某场、某句对白、某段节奏。
- 需要诊断因果、价值转变、人物缺陷、高潮选择。
- 从零创作但已经进入梗概、人物、结构、分场阶段，需要校准骨架。

## 工作纪律

- 不知道就问一个窄问题。
- 只给一个版本，不给多个备选。
- 修改时只改用户要求的范围。
- 不擅自“顺手优化”相邻内容。
- 场景写作前先检查它在结构中的位置。
- 每场写完可附 3-5 行分析。

## 麦基方法

### Controlling Idea 控制性理念

在梗概阶段使用。

格式：

```text
生活变成 X，因为主角做了 Y，而原因是 Z。
```

用途：

- 判断故事是否有脊柱。
- 判断结局是否回应主题。
- 判断场景是否服务同一个核心论断。

如果无法一句话说清，先不要写场景。

### Scene Value 场景价值转变

在分场大纲和场景写作前使用。

每场必须回答：

- 这场开始时，哪个价值处于什么状态？
- 这场结束时，它变成什么状态？
- 是正转负，还是负转正？
- 是哪个动作造成转变？

常见价值对：

- 控制 / 失控
- 信任 / 背叛
- 爱 / 占有
- 自由 / 束缚
- 希望 / 绝望
- 知晓 / 无知
- 团队 / 工具化

没有价值转变的场景，通常是废场或过渡场。

### Inciting Incident 激励事件

在结构大纲阶段检查。

激励事件必须打破主角原有平衡。它之前和之后，主角的世界应当不同。

### Progressive Complications 递进冲突

每一次冲突都应比前一次更难，或代价更高。第三幕冲突不能弱于第一幕。

### Crisis / Climax / Resolution

- Crisis：最后的两难选择。
- Climax：回答这个选择的行动。
- Resolution：高潮之后的新世界坐标。

高潮不是大场面，而是主角做出的最终行动。

### 正负摆动

故事不能一直顺，也不能一直败。每个阶段有正负变化，幅度逐渐变大。

## 坎贝尔方法

坎贝尔用于诊断，不强行套模板。

检查：

- 主角是否有普通世界？
- 是否有召唤？
- 是否拒绝或犹豫？
- 是否跨过门槛？
- 是否经历试炼、盟友、敌人？
- 是否面对最深洞穴或死亡象征？
- 是否带着变化返回？

反英雄或悲剧可以是反向旅程，不要求完整英雄归来。

## 亚里士多德方法

### Hamartia 人物缺陷

人物阶段和剧本医生阶段使用。

人物缺陷不是道德罪恶，而是性格中会导致灾难的错误。它应当：

- 第一场就能看见端倪。
- 在中段不断制造代价。
- 在高潮被彻底引爆。

### Peripeteia 转折

行动导致相反结果。强故事通常有 2-3 次越来越强的转折。

### Anagnorisis 认知

角色或观众发现隐藏真相。最强的认知往往与转折同时发生。

### Catharsis 净化

结局让观众同时感到同情和恐惧：我理解他，也害怕自己可能成为他。

### 行动统一

每场戏都应通向最终点。删掉一场，如果结局不变，它很可能不该存在。

## 因果性审计

在分场大纲、剧集分集、剧本医生阶段使用。

逐场检查：

```text
第 N 场发生，是因为第 N-1 场发生了什么？
```

好结构是“因为”，不是“然后”。

问题标签：

- `[因果]`：这一场不来自上一场的结果。
- `[价值]`：这一场没有价值转变。
- `[人物]`：行为不符合人物缺陷、欲望或前史。
- `[世界观]`：与设定或已公开信息冲突。
- `[节奏]`：过长、过短、重复、信息密度失衡。

## 写场景前检查

写每场前回答：

1. 这场在第几幕或第几阶段？
2. 对应什么结构功能？
3. 进入价值是什么？
4. 退出价值是什么？
5. 它如何服务人物缺陷？
6. 是否是转折、认知，还是铺垫？
7. 如果删掉它，故事会变吗？

回答空洞时，不写，先重构。

## 修改机制

用户给修改意见时：

- 如果只改一句台词，只改这一句。
- 如果只改一个场景目标，不重写整部。
- 如果用户说“不对”，不要立刻给多个版本，先问一个窄问题。

窄问题示例：

```text
这场里，主角是想保护对方，还是利用对方？
```

## 时长与删减

需要压缩时：

1. 先按场景估算实际时长。
2. 找重复情绪、重复信息、过渡场、反应镜头。
3. 不优先删激励事件、转折、认知、危机、高潮。
4. 给具体方案：从哪场删多少秒，为什么。

## 3-5 行场景分析

写完单场后可附：

- 价值转变：进入 -> 退出。
- 人物缺陷：这一场如何暴露或挑战缺陷。
- red flags：因果、价值、人物、节奏是否有隐患。
