# 量化评分表

量化评分是整合技能新增的输出呈现层。它不替代山音自检，也不替代 screenwriter 审计；它只是把诊断结果整理为用户容易判断的 10 分制。

## 使用时机

- 用户要求“诊断”“打分”“看看问题”“能不能拍”“帮我评估”。
- 完成阶段成果后，例如梗概、分场大纲、第一集开篇、短片初稿。
- 剧本医生阶段。
- 从零创作完成一轮后，作为二次审计。

## 输出原则

- 分数后必须有一句简短理由。
- 不只给分，要给修改优先级。
- 不要用分数压倒创作判断；分数是定位问题，不是评判作者。
- 体量不同，评分维度不同。

## 通用诊断格式

```text
【整体判断】
一句到三句，先判断底子和最适合的方向。

【量化评分】
维度：分数/10
理由：一句话。

【主要问题】
1. ...
2. ...
3. ...

【修改优先级】
优先级1：...
优先级2：...
优先级3：...

【下一步】
建议先改哪一项。
```

## 1-3 分钟概念片 / AI 视频 / 微短剧开篇

```text
开场钩子：/10
视觉概念：/10
场景因果：/10
人物主动性：/10
台词/VO克制：/10
结尾钩子：/10
3分钟节奏：/10
```

判断依据：

- 开场钩子：前5-15秒是否抓人。
- 视觉概念：核心规则是否能用画面表达。
- 场景因果：每个段落是否由上一个段落推出。
- 人物主动性：主角是否在做选择，而不是被设定推着走。
- 台词/VO克制：有没有解释概念、解释情绪、解释主题。
- 结尾钩子：是否让观众想看下一条或下一集。
- 3分钟节奏：信息密度是否过满或过空。

## 修改优先级规则

优先改：

1. 因果断裂。
2. 主角没有主动选择。
3. 没有价值转变的场景。
4. 开场不抓人。
5. 结尾没有钩子或余味。
6. 台词解释设定。
7. 节奏过满或过拖。

分数低但不影响主线的项目，放到后面改。
