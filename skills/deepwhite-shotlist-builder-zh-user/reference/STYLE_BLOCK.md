# 风格段落构建规则

本 fork 不使用固定默认风格。不要套用原技能的 Lubezki × Deakins、practicals-only、60:30:10 或固定禁令，除非用户明确要求，或上传资料中能清楚分析出这些规则。

## 风格来源优先级

按以下顺序确定风格：

1. 用户在当前请求中明确说明的风格。
2. 用户上传的风格资料：旧提示词表、导演阐述、look bible、参考图、样片截图、分镜网页、品牌手册、视觉笔记。
3. 剧本自身可分析出的风格信号：题材、年代、地点、人物心理、节奏、场景密度、动作/对白比例、反复出现的视觉母题。

如果三者都不足以确定风格，必须先问用户一个简短问题，例如：

```text
这个项目希望走什么影像风格？比如写实电影、广告质感、港片霓虹、纪录片手持、日系清冷、赛博朋克、古典油画感，或者你自己的描述。
```

## 风格确认

进入正式写提示词前，在中文回复中简短确认风格来源：

```text
我会按你资料里呈现的风格来写：低饱和现实主义、手持近距离观察、冷色自然光、表演克制，重点保留潮湿空气和城市夜景反光。
```

如果风格来自剧本推断，要明确写“我从剧本中推断”，并允许用户修正。

## 四段控制模块格式

每条提示词不再使用后置 `风格：` 段落。风格必须前置到素材句柄之前，并拆成四个固定标题：

这四段是生成控制模块，不是剧情复述。最终内容必须全部用中文写；如果参考资料里出现英文提示词，要翻译为中文控制意图，不要在最终提示词里保留英文标签。

```text
【全局画质】控制整体真实感和生成方向。写清画面偏真实电影摄影、实拍质感、高解析、大画幅、真实物理，还是用户指定的游戏、动画、插画、广告或其他方向；如果用户没有要求游戏/动画感，默认写清不要游戏引擎感、不要三维渲染感、不要插画感。
【人物材质】控制人物近景的真实细节。只写皮肤、头发、眼眶、嘴唇、汗水、水渍、血色、伤口、绒毛、毛孔、次表面散射、身体受力状态等，不重复人物身份、五官、服装全貌或剧情背景。
【灯光与风格】控制灯光方式、色彩倾向、情绪、电影感和高级感。写主光、辅光、负补光、明暗对比、色调、颗粒感、空间反差、参考影片气质等；内容必须来自用户要求、参考资料或剧本分析。
【核心特效】控制本段最有记忆点的视觉内容。如果有特效，具体写特效形态、材质、颜色、运动方式、粒子/流体/能量变化、生成与消散、与人物和环境的物理交互；如果没有超现实特效，就写本段最重要的核心物理视觉机制，例如水下阻力、爆炸烟尘、真实火焰、雨水飞溅、玻璃碎裂、冲击碎屑。
```

不要求每条都机械塞满所有字段。复杂镜头写完整；简单插入镜头保留与画面相关的风格约束即可。四段都要短而硬，优先写会直接影响生成结果的词，不写空泛夸赞。

## 写法示例

```text
【全局画质】真实电影摄影质感，高解析、大画幅、实拍感，画面必须符合真实物理；不要三维渲染感，不要游戏引擎质感，不要动画或插画风格。
【人物材质】女主湿发贴脸，皮肤被冷水浸泡后微微泛白，毛孔和细小绒毛清晰，眼眶因憋气发红，嘴唇湿冷，双手裸露无手套，指节因用力抓握而发白。
【灯光与风格】水下办公室为冷蓝低照度，悬浮颗粒和折射光明显；门外商场为干燥、明亮、暖色商铺灯，白色地面和绿植形成强烈空间反差。
【核心特效】黑色水下龙卷风是高密度流体旋涡，卷起办公桌、椅子、文件和电脑，物体运动有水阻和惯性；门内外形成清晰水体边界，龙卷风不能越过门。
```

## 从资料分析风格

分析资料时优先提取这些要素：

- **影像基调**：写实、超现实、商业广告、纪录片、MV、类型片、短剧、动画化、复古等。
- **色彩**：冷暖、饱和度、对比度、主色/辅色/点缀色、是否有固定品牌色。
- **光线**：自然光、实景灯、霓虹、屏幕光、柔光、硬光、逆光、补光是否可见。
- **摄影机**：手持、固定、推拉、摇移、长镜头、快速剪辑、广角空间、长焦压缩。
- **质感**：颗粒、胶片、高清数字、手机影像、监控影像、湿润空气、灰尘、烟雾。
- **表演尺度**：自然主义、强戏剧化、压抑克制、喜剧节奏、惊悚反应。
- **声音**：纯环境声、音乐驱动、无音乐、对白优先、环境噪声主导。
- **禁令**：用户资料中明确不想要的元素。
- **代表作**：如果用户提到电影、剧集、广告或艺术家作品名，可在【全局画质】中写入作为风格锚点。

## 灯光规则

灯光不再固定为 practicals-only。按风格来源决定：

- 如果用户要求自然主义或实景光，就强调真实光源、方向和限制。
- 如果用户要求广告、MV、舞台、时尚大片等风格，可以允许补光、造型光、彩色光或非现实光源。
- 如果资料里没有灯光规则，就根据场景和情绪推断，并在提示词中写明“灯光依据场景推断”。

无论风格如何，都要写清：

- 主光来自哪里
- 辅助光是否存在
- 人物脸部哪一侧受光
- 背景光和人物皮肤是否互相污染
- 屏幕、霓虹、车灯、窗光、台灯等是否会成为主光源

## 色彩规则

不再固定使用 `60:30:10`。只有在用户资料或你对资料的分析中适合时才使用。

可选写法：

```text
色彩：低饱和冷灰为主，暖黄灯光只作为局部点缀。
色彩：高对比霓虹，洋红和青色对撞，黑色阴影占画面大面积。
色彩：接近纪录片自然色，肤色不过度美化，环境色轻微偏绿。
```

## 禁令规则

不要无条件加入固定禁令。只写与当前风格和镜头相关的禁令。

常见可选禁令：

```text
禁身份漂移。
禁漂浮道具。
禁字幕。
禁背景音乐。
禁过度磨皮。
禁游戏CG质感。
禁焦点漂移。
禁空间左右颠倒。
```

如果用户明确想要游戏CG、动画、强霓虹、舞台补光或字幕，不要用默认禁令抵消用户风格。

## 单镜头 / 多镜头开头

多镜头提示词开头：

```text
多机位剪辑。
```

单镜头提示词开头：

```text
单镜头，无剪辑。
```

## 不使用固定结尾

不要把 `15秒。21:9。` 放在每条提示词最后。

如果需要记录时长或画幅，把它们放在网页中的“时长”“画幅”字段，或在镜头段落内部作为具体时长说明。不要把它作为每条提示词的统一收束句。
