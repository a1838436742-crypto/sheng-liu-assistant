# 空间确认 Gate

任何重要人物关系、多人互动、关键道具位置、摄影机几何关系，都必须先确认空间。空间确认不是可选优化，而是提示词写作前的硬性 gate。

## 必须画图的情况

以下任一条件成立，就必须画俯视调度图：

- 场景中有两个或更多故事角色参与互动，即使某些镜头不是同框
- 两个角色之间有距离、遮挡、追逐、对峙、拥抱、递交、打断、注视等关系
- 两个或多个主要内容之间存在前后、左右、面前、背后、遮挡、靠近、远离、上下层级等关系
- 关键道具需要放在具体表面或被具体角色拿取
- 摄影机从某个明确方向拍摄会影响叙事
- 用户要求调整站位、朝向、距离、道具位置或镜头方向
- 视频模型容易左右颠倒、前后错位、把人物并排放错

可以跳过的情况：

- 单人无关键道具的纯表情特写
- 无人物关系的纯环境镜头
- 无位置要求的抽象插入镜头

跳过时必须写明原因，例如：

```text
本次范围无需位置图：场景12只有单人脸部特写，没有关键道具落点，也没有人物关系。
```

## 必须输出的空间确认格式

用中文输出：

```markdown
## 位置关系确认：场景 {N}

我先锁定人物、道具和摄影机位置。请检查后回复 `确认位置`，或告诉我怎么改。

**主视角轴线**
- 北：窗户 / 巨型屏幕 / 舞台深处
- 南：入口 / 走廊 / 摄影机初始方向
- 东：厨房 / 桌子右侧
- 西：沙发 / 墙面照片

**人物与道具**
- A / Roko：站在客厅南侧入口内约0.5米，面朝北侧冰箱。
- B / Lulu：坐在西侧沙发，距Roko约2.3米，面朝东南。
- P1 / 拍立得：贴在冰箱门右上角，距地约1.5米。

**主要相互位置关系**
- Roko在Lulu的东南方向，距离约2.3米；Lulu在Roko左前方。
- 冰箱在Roko正前方偏北，拍立得贴在冰箱门右上角。
- Roko背后是南侧入口，Lulu背后是西侧沙发靠背。
- 镜头2中Roko侧脸会遮挡走廊背景的一部分。

**摄影机位置**
- 镜头1：C1在入口南侧，朝北拍Roko进门。
- 镜头2：C2在冰箱东南侧，85mm拍Roko脸部反应。
- 镜头3：C3贴近冰箱门，拍P1大特写。

**关键限制**
- ⚠️Roko不能站到Lulu旁边。
- ⚠️Lulu不能出现在冰箱旁。
- ⚠️拍立得必须在冰箱门右上角，不能漂浮，必须贴在门面。
- ⚠️必须保持前后关系：冰箱在Roko面前，入口在Roko背后，Lulu在Roko左前方。
```

随后给出一个俯视 SVG 或清楚的 ASCII / Mermaid / diagram 示意。环境支持可视化工具时优先用可视化图；没有工具时，用文本俯视图也可以，但必须足够明确。

结束时必须问：

```text
位置对吗？请回复 `确认位置`，或告诉我哪一项要调整。
```

## 停机规则

输出空间确认后必须停止。不得同时输出提示词或 HTML。

只有当用户明确回复以下之一时，才能继续：

- `确认位置`
- `位置确认`
- `位置对`
- 其他明确表示空间关系正确的话

如果用户改位置，必须更新空间确认图并再次等待确认。

## 写入提示词

确认后，每条相关提示词必须包含 `⚠️空间布局`，并忠实反映最新版确认图：

```text
⚠️空间布局（已确认）：主轴线为南侧入口看向北侧冰箱。@image1站在入口内约0.5米，面朝北；@image2坐在西侧沙发，距@image1约2.3米，面朝东南；@image3贴在冰箱门右上角，距地约1.5米。主要相互位置关系：冰箱在@image1面前，入口在@image1背后，@image2在@image1左前方，走廊背景在@image1背后并被侧脸部分遮挡。⚠️禁止把@image2移到冰箱旁，禁止把@image3变成漂浮道具。
```

## 进入提示词前的状态行

空间确认后，在下一步开头写：

```text
已确认位置：是
```

如果某场景无需位置图，写：

```text
已确认位置：本次范围无需位置图，原因：...
```
