# 故事板资产规则

故事板是可选资产。它通常是一张或多张带编号的图片，包含分镜图、文字说明、动作说明、镜头编号、构图信息或台词框。故事板用于理解构图和动作，不是最终画面的一部分。

## 资产准备阶段必须询问

在资产请求阶段，必须问用户：

```text
是否需要使用故事板/分镜图参考？如果需要，请上传带编号的故事板图片。故事板可以包含分镜图、镜头编号、动作说明和文字信息。
```

如果用户不需要故事板，记录：

```text
故事板：不使用。
```

## 资产确认表

故事板要进入资产确认表：

```markdown
| 文件名 | 我识别为 | 类型 | 用途场景 | 状态 | 备注 |
|---|---|---|---|---|---|
| storyboard_01.png | SB-01 至 SB-04 | 故事板 | 场景3 | 已匹配 | 含4个编号分镜和文字说明 |
| storyboard_02.png | SB-05 至 SB-08 | 故事板 | 场景3 | 需要用户确认 | 编号较小，需要确认是否连续 |
```

如无法看清编号，必须问用户，不要自行编造。

## 提示词中的引用方式

如果使用故事板，每条提示词必须说明使用哪些故事板编号：

```text
故事板参考：使用 SB-03、SB-04 的构图、人物站位和动作信息。
```

每个镜头也要写：

```text
故事板参考：SB-03。
```

如果一个提示词或镜头没有使用故事板，完全省略故事板相关内容。不要写 `故事板参考：本条不使用故事板。`、`故事板参考：无。`、`未使用故事板` 或任何等价占位说明。

## 禁止故事板痕迹进入画面

只要使用故事板，提示词中必须写：

```text
⚠️故事板仅作为构图、动作和信息参考，最终视频画面中禁止出现故事板编号、标注、线段、箭头、分镜框、文字说明、台词框、注释框、水印、页边框。
```

这条可以放在故事板参考之后，也可以放在必要防错警告中；但必须出现。

如果没有使用故事板，不要写这一条警告，也不要出现故事板编号、故事板参考、分镜图参考等相关标签。

## 使用原则

- 参考故事板里的构图、人物位置、动作方向、镜头编号、文字说明。
- 不复制故事板的图纸质感，除非用户明确要“故事板风格”。
- 不把故事板里的箭头、框线、编号、注释渲染到最终画面。
- 如果故事板和用户最新文字要求冲突，用户最新文字要求优先。
- 如果故事板和确认后的空间图冲突，先向用户确认，不要自行决定。
