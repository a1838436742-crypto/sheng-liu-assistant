# 提示词密度

本文件说明如何把镜头行组合成提示词组。每条提示词最多对应15秒视频；但不要在提示词结尾固定追加 `15秒。21:9。`。每条提示词组的实际时间段、预计时长、提示词数量必须先按 `PROMPT_TIMING.md` 提出方案并让用户确认。

## 基本规则

平均可以理解为 **约4-5个镜头行组成1条提示词**，但不同场景差异很大。不要机械套比例，优先保证整体观看节奏：
- 场景21：7行 / 5条提示词，密度很高，几乎一个节拍一条
- 场景23：3行 / 1条提示词，合并为一个连续情绪单元
- 场景28：42行 / 约9条提示词，动作场面常见密度

**没有固定比例。按场景需要决定。**

## 组合判断

同时满足以下条件时，可以组合成一条提示词：
1. 同一组角色
2. 同一地点或同一地点子区域
3. 情绪和时间连续，没有跳时，没有重大情绪转向
4. 可以在一个短生成单元内完成，通常不超过约15秒
5. 中文提示词不会长到难以实际生成，通常不超过约2500字
6. 内部镜头时长可以被清楚分配，且加总后形成明确的预计时长
7. 合并后不会超过15秒

出现以下任一情况时，拆成独立提示词：
1. 地点硬切，例如公寓到回忆
2. 主要角色进出导致句柄列表变化
3. 景别或镜头设置大幅变化，例如广角建立到道具大特写
4. 表演弧线需要独立承载，例如7个情绪节拍构成的反应特写
5. 道具、屏幕、手部等插入镜头，通常单独写大特写提示词
6. 合并后会超过15秒

## 提示词内部的多镜头剪辑

一条提示词可以包含内部 `【镜头1】 / 【镜头2】 / 【镜头3】`，前提是地点、角色和情绪单元保持连续。语法见 [PROMPT_PATTERNS.md](PROMPT_PATTERNS.md)。

适合多镜头：
- 一个连续情绪瞬间里的2-3个快速切换，例如广角建立、特写反应、低角度收尾
- 情绪统一的小蒙太奇，例如照片扫描、落到关键文字、手伸入拿走

适合单镜头：
- 一个连续表演瞬间，只有一个摄影机运动
- 情绪重量需要完整落下，不应被剪切打断

## 时间安排

拆分提示词时，同步安排时间：

```text
提示词1：00:00-00:04，约4秒
提示词2：00:04-00:10，约6秒
提示词3：00:10-00:18，约8秒
```

时间段按场景内重新计时。每条提示词都要有预计时长，多镜头提示词还要有内部镜头时长。详细规则见 [PROMPT_TIMING.md](PROMPT_TIMING.md)。

在正式写提示词前，必须把这个时间安排表发给用户确认。用户回复 `确认时间划分` 后才能继续。

## 示例

**场景23，3行 → 1条提示词**
- 行11.1：Roko在公寓里，气氛黑暗压抑
- 行11.2：眼泪落下
- 行11.3：闭上眼睛，崩溃哭出声

这三行是厨房地面上的一次连续情绪崩塌。可以写成一条提示词，内部用 `【镜头1】【镜头2】` 分节。**不要把悲伤切碎。**

**场景21，7行 → 5条提示词**
- 提示词1：开门 + 靴子跨过门槛
- 提示词2：从走廊走向客厅，这段需要自己的呼吸节奏
- 提示词3：客厅扫视 + 窗户 + 转向冰箱，50mm单镜头
- 提示词4：冰箱大特写，拍立得滑出 + 手伸入
- 提示词5：照片近景 + Roko脸部反应 + 转身

五个不同摄影设置、焦段和情绪微节拍，应该拆成五条提示词。

## 不确定时

倾向于 **少而准**：只要不超过15秒，且角色、地点、情绪、摄影逻辑连续，就优先合并。不要把一个流畅表演切成很多零碎提示词。Seedance 需要清晰提示词，但整体剧情也需要连贯节奏。

短提示词使用限制：
- 1-3秒提示词只用于闪现、插入、转场、关键道具特写。
- 不要连续出现多条1-3秒提示词，除非这是有意的蒙太奇或动作剪辑。
- 连续反应、凝视、压抑、崩溃、犹豫等表演段落，优先做成8-15秒完整提示词。

## 标签

每条提示词在 HTML 里用一个中文短标签说明画面内容。示例：
- `[中景到特写 · 开门 + 湿靴入屋]`
- `[大特写 · 冰箱照片滑出 + 伸手拿取]`
- `[特写 · Roko脸部反应 + 转身]`
- `[广角到近景 · 空间建立 + 反应]`
- `[多镜头 · 三个反应 + 对白]`

内部景别代码见 [PLAN_TYPES.md](PLAN_TYPES.md)，网页可见标签必须是中文。
