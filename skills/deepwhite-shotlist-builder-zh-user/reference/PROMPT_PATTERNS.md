# 提示词结构规则

每条 Seedance 2.0 提示词都用中文写。提示词不是复述剧本，而是把剧本节拍翻译成摄影、构图、调度、表演、灯光、音效、背景和防错约束。

## 写作前置条件

写任何提示词前必须已经完成：

```text
已确认资产：是
已确认位置：是 / 本次范围无需位置图，原因：...
已确认风格：是
已确认时间划分：是
已确认提示词结构：是
已确认交付形式：生成网页 / 只要文字提示词
```

如果缺少任一状态，停止写提示词，回到对应 gate。不要“先写一版”。

## 固定结构顺序

每条最终可复制的视频提示词，按以下顺序写，不要跳过，不要改序：

1. **固定开头**：`不要出现BGM，不要出现字幕`
2. **风格四段**：`【全局画质】`、`【人物材质】`、`【灯光与风格】`、`【核心特效】`
3. **素材句柄声明**：`@image1`、`@image2` 等
4. **故事板参考（条件字段）**：只有本条实际使用故事板时才出现，说明故事板编号和禁止故事板痕迹进入画面；如果不用故事板，完全省略这一项，不写任何“本条不使用故事板”“无故事板”之类的占位说明
5. **空间与镜头数量警告**：保留空间布局和镜头数量；删除独立 `时间安排：`；删除 `⚠️对白规则：`
6. **镜头段落**：`【镜头1】`、`【镜头2】`，或单镜头段落
7. **环境活动 / 全场音效**
8. **必要防错警告**

不要在每条提示词最后固定追加 `15秒。21:9。`。时间安排只出现在网页时间列、时间划分方案、或镜头标题内部，不作为提示词正文的独立段落。

## 提示词长度控制

每条最终可复制的视频提示词正文必须控制在 2200 个中文字符以内，目标范围为 1800-2200 字。这里的“正文”指用户复制去生成视频的提示词块，不包括网页中的时间列、资产图对照表、字数统计、可删减建议或备注。

如果草稿超过 2200 字，必须先压缩再输出。压缩顺序：

1. 删除重复的风格形容词和空泛夸赞。
2. 合并重复的背景环境描述。
3. 缩短非关键音效。
4. 减少普通 `⚠️` 防错项，只保留最容易影响生成结果的关键防错。
5. 保留素材句柄、四段风格控制、核心动作、画面构图、机位、关键特效、关键空间关系和每镜头音效。

如果生成网页，在提示词块外显示：

```text
字数：约 XXXX / 2200
可手动删减：可优先删减……（不进入复制提示词正文）
```

## 1. 固定开头

每条提示词第一句必须是：

```text
不要出现BGM，不要出现字幕
```

这句话必须位于所有素材句柄和风格描述之前。

## 2. 风格四段

固定开头后，立即写四段风格信息。不要再把风格段落放到镜头段落后面。

格式：

```text
【全局画质】控制整体真实感和生成方向。写清画面偏真实电影摄影、实拍质感、高解析、大画幅、真实物理，还是用户指定的游戏、动画、插画、广告或其他方向；如果用户没有要求游戏/动画感，默认写清不要游戏引擎感、不要三维渲染感、不要插画感。
【人物材质】控制人物近景的真实细节。只写皮肤、头发、眼眶、嘴唇、汗水、水渍、血色、伤口、绒毛、毛孔、次表面散射、身体受力状态等，不重复人物身份、五官、服装全貌或剧情背景。
【灯光与风格】控制灯光方式、色彩倾向、情绪、电影感和高级感。写主光、辅光、负补光、明暗对比、色调、颗粒感、空间反差、参考影片气质等；内容必须来自用户要求、参考资料或剧本分析。
【核心特效】控制本段最有记忆点的视觉内容。如果有特效，具体写特效形态、材质、颜色、运动方式、粒子/流体/能量变化、生成与消散、与人物和环境的物理交互；如果没有超现实特效，就写本段最重要的核心物理视觉机制，例如水下阻力、爆炸烟尘、真实火焰、雨水飞溅、玻璃碎裂、冲击碎屑。
```

风格内容必须来自用户说明、上传资料或剧本分析。不要套用固定导演名或固定灯光规则，除非用户资料明确要求。最终提示词中这四段必须使用中文，不要保留 `Photorealistic`、`Texture`、`Lighting`、`Visual Style`、`VFX` 等英文标签；如果参考资料里有英文提示词，要翻译成中文控制意图。

## 3. 素材句柄声明

每条提示词都要重新声明本条用到的素材。句柄按每条提示词重新编号，不按全片固定。

示例：

```text
@image1（Roko）——亚欧混血，身高约178cm，偏瘦但肩背紧实，深色凌乱中长发，鼻梁红色创可贴，左颊小痣，黑色贴身T恤，黑工装裤，黑靴，右腰挂红色小熊挂件，左前臂纹身。当前状态：发梢微湿，脸上有薄水雾。
@image2（旧公寓）——三视图参考：上图=客厅朝走廊方向，下图=窗边沙发区，中图=厨房入口；红色电视墙，两扇大窗，窗外城市夜景。
@image3（拍立得照片）——横版拍立得，约11cm×8.5cm，自拍合照，底部手写“NOV 14”。
```

规则：

- 人物句柄写清身高、体型、发型、脸部识别点、从头到脚服装、随身道具。
- 场景之间有湿、干、血、灰尘、破损等状态变化时，必须在句柄里写明。
- 场景句柄写清参考图布局；多视图参考要说明上图、下图、中图分别是什么。
- 道具句柄写清尺寸、文字内容、颜色、材质、摆放状态。
- 静态姿势参考图只能用于姿势，必须警告不得渲染成视频画面。

## 网页资产图对照表

当最终交付为 HTML 网页时，每条提示词组都必须在复制提示词正文外增加“本段使用资产图对照表”。表格只解释本段拍摄用到哪些资产，不进入可复制提示词正文。

表格字段固定为：

```text
句柄｜资产名称｜原始文件 / 图号｜类型｜本段用途｜使用镜头｜注意事项
```

填写规则：

- 只列本条提示词实际使用的资产图，不列无关全片资产。
- `句柄` 必须与本条提示词正文里的 `@image1`、`@image2` 等一致。
- `本段用途` 写清该图在本段中控制什么，例如人物身份、服装、场景布局、道具形态、构图参考、动作参考。
- `使用镜头` 写 `镜头1`、`镜头1-3` 或 `全段`。
- 如果没有使用故事板，不要在资产对照表里出现故事板行。
- 如果使用故事板，故事板也作为资产行出现，并在注意事项里写清“只参考构图/动作/文字信息，最终画面禁止出现编号、标注、线段、台词框”。

## 4. 故事板参考

如果使用故事板，每条提示词必须说明使用哪些故事板编号：

```text
故事板参考：使用 SB-03、SB-04 的构图、人物站位、动作方向和文字说明。
⚠️故事板仅作为构图、动作和信息参考，最终视频画面中禁止出现故事板编号、标注、线段、箭头、分镜框、文字说明、台词框、注释框、水印、页边框。
```

每个镜头内部也要写 `故事板参考：SB-xx`，但前提是该镜头实际使用了故事板。

如果本条提示词或某个镜头没有使用故事板，必须完全省略故事板相关内容。不要写 `故事板参考：本条不使用故事板。`、`故事板参考：无。`、`未使用故事板` 或任何等价占位说明；也不要写“禁止故事板痕迹进入画面”的警告，因为没有故事板时不需要触发这一类防错。

## 5. 空间与镜头数量警告

镜头段落前必须声明空间和镜头数量。不要写独立的 `时间安排：` 段落。不要写 `⚠️对白规则：`。

```text
⚠️空间布局 / 人物位置：引用已确认的俯视调度图，写清人物、道具、摄影机位置、距离、朝向、遮挡关系。
⚠️本视频严格只有N个镜头——禁止添加额外镜头，禁止自动补镜头。
```

对白仍可在具体镜头的动作和微表演中处理，但不要作为全局规则块出现。

## 6. 多镜头提示词

多镜头提示词开头写：

```text
多机位剪辑。
```

每个内部剪辑使用一个 `【镜头N】` 段落。每个镜头里，`机位：` 前必须先写 `画面动作概述：` 和 `画面构图：`。如果该镜头实际使用故事板，则在 `画面动作概述：` 前增加 `故事板参考：SB-xx`；如果没有使用故事板，不要出现任何故事板字段。

格式：

```text
【镜头1｜0.0-0.5秒】
故事板参考：SB-03。
画面动作概述：Roko推门进入，身体僵硬警觉，湿靴跨过门槛，右手仍握着照片，空间关系被瞬间建立。
画面构图：初始画面为Roko位于画面左三分之一，前景偏中景，身体中心约在x=32%，占画面高度约45%；门框压在画面最左侧形成前景边框，客厅深处在右侧中远景形成负空间。Roko在门框前方，客厅环境在他背后延伸，照片固定在右手手边，摄影机移动时保持人物左侧、客厅右侧、门框前景遮挡的关系。
机位：摄影机位于角色南侧偏东，35mm广角，全景，轻微手持推进；持续时间严格约0.3-0.5秒，只用于瞬间建立空间。
动作：门打开，湿靴跨过门槛，右手仍握着照片，肩膀微微上提，眼神先扫向客厅深处再落回手中照片。
音效：门轴轻响、湿靴踩过门槛的闷声、远处城市低频嗡鸣。

【镜头2｜0.5-6.0秒】（紧接镜头1硬切而来）
故事板参考：SB-04。
画面动作概述：Roko停在走廊阴影里，低头看向照片边缘，身体仍绷紧，情绪从警觉转为迟疑。
画面构图：初始画面为Roko侧脸占画面左中部，前景特写，脸部中心约在x=47%、y=42%；照片在他面前下方，靠近画面右下区域，走廊暗部在背后中远景虚化。照片在Roko面前，走廊背景在他背后，侧脸遮挡部分灯影，摄影机微动时必须保持照片位于人物面前下方。
机位：摄影机贴近Roko右前侧，85mm长焦，F1.4极浅景深，侧脸紧凑特写，手持呼吸式微动；禁稳定器，焦点锁在眼睛和照片边缘之间。
动作：①他停住；②眼睛向照片下缘移动；③喉结吞咽；④右手指尖收紧照片边缘；下颌肌肉缓慢收紧，眼眶湿润但不落泪。
音效：纸质照片被手指压弯的细微声、角色压住呼吸的短促鼻息。
```

每个镜头段落必须使用下面固定字段，不要额外拆出 `摄影机运动：`、`摄影机位置：`、`背景：`、`微表演细节：` 等独立字段；这些内容需要时合并写入 `机位：` 或 `动作：`。

- **故事板参考（条件字段）**：只有本镜头实际使用故事板时才写本镜头使用的故事板编号；没有使用故事板时完全省略，不写“无”或“本条不使用故事板”。
- **画面动作概述**：在 `机位：` 前，用一句话写这个镜头看见什么、主要动作、人物状态、情绪/叙事功能。
- **画面构图**：写初始画面构图。优先使用左三分之一、中央、右三分之一、前景、中景、远景、背景、负空间、遮挡、接触点和相互位置关系。百分比只作为可选辅助锚点，不是数学保证。若使用百分比，坐标为屏幕坐标：x=0%左边缘，x=50%中心，x=100%右边缘；y=0%顶部，y=50%中心，y=100%底部。
- **机位**：合并写摄像机位置、景别、镜头角度、焦段、运镜方式，并说明运镜过程中必须保持的构图关系。
- **动作**：按时间顺序写人物动作、物体运动、特效变化和必要微表演，不要抽象概括。
- **音效**：每个镜头必须写。若是贯穿全场的声音，在本镜头可写“延续全场音效”，并在环境活动 / 全场音效中详细说明。

镜头内部固定字段顺序：

```text
画面动作概述：...
画面构图：...
机位：...
动作：...
音效：...
```

## 7. 单镜头提示词

单镜头提示词开头写：

```text
单镜头，无剪辑。预计时长约X秒。
```

单镜头同样必须在 `机位：` 前写 `画面动作概述：` 和 `画面构图：`。如果实际使用故事板，则在前面增加 `故事板参考：SB-xx`；如果没有使用故事板，完全省略故事板行。

```text
画面动作概述：一句话概括这个连续镜头的主要动作、人物状态、情绪变化、叙事目的。
画面构图：写初始构图、画面区域、景深层级、主体占画面大小、接触点、负空间、前后左右、面前/背后、遮挡、距离等相互关系；百分比坐标只作为可选辅助锚点。
机位：写摄像机位置、景别、镜头角度、焦段、运镜方式，并说明运镜过程中保持哪些构图关系。
动作：按时间顺序写连续动作、微表演和特效变化。
音效：写本镜头声音。
```

并且必须包含 `音效：...`。

## 8. 空间布局写法

任何两个以上角色同框、关键道具落在具体表面、复杂摄影机几何关系，都必须写空间布局。内容必须来自已确认的俯视调度图。

必须写：

- 米制距离
- 相对方位或东西南北
- 主要内容的相互位置关系：人物在谁面前/背后，谁在左侧/右侧，谁靠近/远离，谁遮挡谁
- 谁看向谁
- 谁遮挡谁
- 道具在具体桌面/地面/墙面的位置
- 模型容易搞错的高度、眼线、左右关系

## 9. 对白处理

删除全局 `⚠️对白规则：` 段落后，对白规则下沉到具体镜头中处理。

如果本镜头有对白，在该镜头 `动作：` 中写清：

```text
台词前：一次短促鼻吸气，喉结吞咽。
台词：“别再问我。”——重音落在“再”，视线严格朝向@image2。
台词后：视线继续锁定对手0.5秒，然后微微向下移开。
```

如果角色A打断角色B，也写在具体镜头里，不写全局对白警告。

## 10. 环境活动 / 全场音效

有人群或工作环境时，背景必须有具体行为，禁止空旷化。

如果某个音效贯穿全场景，在这里描述：

```text
环境活动 / 全场音效：背景工作人员持续低声交谈、键盘敲击声贯穿全场，远处设备保持低频嗡鸣；每个镜头中的近景音效在各镜头单独说明。
```

安静场景也要写清安静的内容：

```text
环境活动 / 全场音效：完全寂静，仅有远处城市低频声、暖气管道轻响、湿靴踩在地板上的回响。
```

## 11. 必要防错警告

预判 Seedance 最可能出错的地方，用 `⚠️` 标记。关键错误用 `⚠️⚠️⚠️`。

常见警告：

- 身份漂移：每个角色服装和脸部识别点保持一致。
- 句柄污染：禁止把A角色的衣服、发型、道具套到B角色身上。
- 道具错位：道具必须落在指定桌面/手里/墙上。
- 漂浮道具：道具必须有接触阴影和重量。
- 多加镜头：本视频严格只有N个镜头。
- 焦点漂移：插入镜头焦平面必须锁定目标。
- 空间镜像：禁止左右颠倒、前后错位。
- 背景空掉：有人群或工作环境的场景必须保留背景活动。
- 故事板污染（仅在实际使用故事板时）：禁止出现故事板编号、标注、线段、箭头、分镜框、文字说明、台词框、注释框、水印、页边框。

插入镜头标准焦点锁定：

```text
⚠️焦平面从第一帧到最后一帧严格锁定在[物体]上——绝对禁止焦点漂移、禁止自动跳焦、禁止中途换焦。
```
