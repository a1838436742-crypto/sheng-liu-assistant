# 提示词时间划分与确认

每条最终拆分出来的提示词都必须有明确时间安排。这里的时间安排是网页生产信息和镜头调度信息，不是固定写在提示词最后的收束句。

更重要的是：**一段剧情如何划分时间、会生成几条视频提示词，必须先和用户沟通确认。**

## 时间划分确认 Gate

在写任何正式提示词或生成 HTML 前，必须先输出“时间划分方案”，并等待用户确认。

必须询问用户确认：

```text
请确认这个时间划分和提示词数量。回复 `确认时间划分` 后，我再开始写正式提示词和网页。
```

用户未确认前，停止。不要继续写提示词。

## 时间划分方案格式

用中文输出：

```markdown
## 时间划分方案

我先把这段剧情按视频生成节奏拆分。每条提示词最多对应15秒视频；为了整体节奏，我会优先合并连续剧情，避免切得太碎。

| 提示词 | 剧情段落 | 场景内时间段 | 预计时长 | 内部镜头 | 为什么这样分 |
|---|---|---:|---:|---|---|
| 1 | 开门进入，建立空间和人物状态 | 00:00-00:08 | 约8秒 | 镜头1 1秒；镜头2 7秒 | 同一地点、同一情绪进入动作，合并更顺 |
| 2 | 发现照片，手伸向冰箱 | 00:08-00:15 | 约7秒 | 镜头1 2秒；镜头2 5秒 | 道具成为焦点，独立成段 |
| 3 | 看清照片后的脸部反应 | 00:15-00:27 | 约12秒 | 单镜头12秒 | 情绪弧线完整，不拆碎 |

总计：约27秒，3条视频提示词。

请确认：回复 `确认时间划分`，或告诉我哪几段要合并/拆开。
```

## 分段原则

每条提示词最多对应15秒视频。这是上限，不是目标。不要把每条都硬凑15秒。

为了总体节奏，禁止过度拆分：

- 能在一个连续情绪/动作单元里完成的内容，优先合并。
- 连续地点、连续角色、连续情绪、连续摄影逻辑，可以合并到一条提示词中。
- 只有在超过15秒、焦点彻底改变、地点硬切、角色进出、关键道具插入、情绪大转折时，才拆新提示词。
- 1-3秒的短提示词只用于闪现、道具插入、过渡、极短建立镜头；不要连续堆很多短提示词。
- 反应戏、悲伤崩溃、压抑情绪、凝视等完整表演弧线，优先保留为一条8-15秒提示词。

## 拆分上限判断

如果一段剧情被拆成太多提示词，必须主动自检：

```text
这些提示词是否只是因为镜头变化被切开？如果角色、地点、情绪没有变化，能否合并？
```

推荐节奏：

- 10秒以内的简单剧情：通常1条提示词
- 10-25秒的连续剧情：通常2-3条提示词
- 25-45秒的完整场景：通常3-5条提示词
- 动作密集、道具多、角色多的场景：可以更多，但每次拆分都要有明确理由

不要为了“更细”而把每个镜头都变成一条提示词。

## 必须输出的时间字段

每条提示词组都要有：

- **场景内时间段**：例如 `00:00-00:07`
- **预计时长**：例如 `约7秒`
- **内部镜头时长**：例如 `镜头1 0.5秒；镜头2 4秒；镜头3 2.5秒`
- **生成建议**：例如 `建议生成7秒`、`建议生成10秒`、`建议生成15秒以内`

在 HTML 表格中显示为独立的“时间安排”列。不要只把时长藏在提示词正文里。

## 时间范围规则

每个场景从 `00:00` 开始重新计时。不要跨场景累计，除非用户明确要求全片连续时间码。

示例：

```text
场景21
提示词1：00:00-00:04，约4秒
提示词2：00:04-00:10，约6秒
提示词3：00:10-00:18，约8秒
```

如果同一场景内有跳时、回忆或幻想，要在时间字段里标注：

```text
00:18-00:24（回忆闪回）
00:24-00:29（回到现实）
```

## 单条提示词时长建议

每条绝对不要超过15秒，但不必每条都做成15秒。

常用范围：

- 道具插入 / 闪现建立：1-3秒
- 一句对白：3-7秒
- 简单动作：4-8秒
- 无台词反应：5-10秒
- 完整情绪弧线：8-15秒
- 多镜头动作小段：8-15秒

如果用户使用的视频工具固定只能生成某个时长，可以把“预计时长”改成该工具需要的时长，并在“内部镜头时长”里分配节奏；但单条仍不能超过15秒，除非用户明确说明该工具和项目允许更长。

## 多镜头提示词内部时间

多镜头提示词必须在每个 `【镜头N】` 标题或第一行写内部时长：

```text
【镜头1｜0.0-0.5秒】
机位：35mm广角，全景。只用于瞬间建立空间。

【镜头2｜0.5-5.5秒】
机位：85mm长焦，F1.4极浅景深，Roko脸部特写。

【镜头3｜5.5-8.0秒】
机位：50mm手部近景，焦点锁定拍立得边缘。
```

单镜头提示词也要写：

```text
单镜头，无剪辑。预计时长约8秒。
```

或：

```text
【单镜头｜00:00-00:08】
```

## 网页标签写法

每条提示词标题旁写短时间标签：

```html
<span class="prompt-time">00:04-00:10 · 约6秒</span>
```

“时间安排”列写完整拆分：

```text
场景内：00:04-00:10
预计：约6秒
内部：镜头1 0.5秒；镜头2 5.5秒
生成：建议生成6秒或向上取整到工具可选时长
```

## 和“不固定收尾”的关系

禁止把 `15秒。21:9。` 当作每条提示词最后一句。

允许这样写：

- HTML 时间列：`预计约8秒`
- 提示词标题：`提示词3｜00:10-00:18｜约8秒`
- 镜头段落：`【镜头2｜1.0-6.0秒】`

不允许这样写：

```text
... 15秒。21:9。
```

作为每条提示词固定结尾。

## 自检

交付前检查：

- 每条提示词组是否都有时间段
- 每条提示词组是否都有预计时长
- 每条提示词是否都不超过15秒
- 是否已经向用户确认提示词数量和时间划分
- 多镜头提示词内部时长加总是否等于该提示词预计时长
- 场景内提示词时间段是否连续或明确标注跳时
- 时间安排是否显示在 HTML 中，而不是只写在提示词正文里
- 是否存在可以合并的连续短提示词；如有，先合并再给用户确认
