# HTML 布局 QA Gate

交付前必须检查网页布局。不要只生成 HTML 就交付。

## 结构规则

生成的 HTML 必须满足：

- 不使用 `rowspan`
- 不使用 `colspan`
- 一条提示词组对应一行 `.prompt-row`
- 每条 `.prompt-row` 必须有独立“时间安排”单元格
- 每条 `.prompt-row` 必须在提示词单元格内包含“本段使用资产图对照表”
- 每条 `.prompt-row` 必须显示可复制提示词正文字数，且 `.prompt-block` 中的提示词正文不超过 2200 字
- “可手动删减建议”必须在 `.prompt-block` 外，不得被复制按钮一起复制
- 多个镜头行放在同一个单元格内，用 `.shot-line` 纵向堆叠
- 提示词放在 `.prompt-block`，使用 `white-space: pre-wrap` 和 `overflow-wrap: anywhere`
- 表格使用 `table-layout: fixed`
- 所有单元格 `vertical-align: top`
- 长文件名、长中文句子、长提示词不能撑破表格

## 必须检查的错位类型

交付前检查：

- 表头和列内容是否对齐
- 提示词是否覆盖动作列或剧本列
- 长中文是否溢出容器
- 复制按钮是否压住标题或提示词
- 手机宽度下是否可以横向滚动，而不是挤压重叠
- 打印/PDF 时是否仍保持基本可读
- 筛选和搜索是否不会隐藏错误行
- 资产图对照表是否没有撑破提示词列
- 字数标签是否显示为 `约 XXXX / 2200 字`
- 可手动删减建议是否位于复制提示词正文外

## 推荐检查方式

优先使用浏览器渲染或截图检查。若不能打开浏览器，至少做静态检查：

```text
1. 搜索 HTML 中是否存在 rowspan 或 colspan；若存在，改掉。
2. 确认 CSS 中存在 table-layout: fixed。
3. 确认 .prompt-block 有 white-space: pre-wrap; overflow-wrap: anywhere; word-break: break-word。
4. 确认 .table-wrap 有 overflow-x: auto。
5. 确认每一条 <tr class="prompt-row"> 的直接 <td> 数量都是6个。
6. 确认每条提示词都有时间段、预计时长和内部镜头时长。
7. 确认每条提示词预计时长不超过15秒。
8. 确认 HTML 使用的是用户已确认的时间划分和提示词数量。
9. 确认每条提示词组都有“本段使用资产图对照表”。
10. 确认每条 `.prompt-block` 中的可复制提示词正文不超过 2200 字。
11. 确认 `可手动删减` 文本不在 `.prompt-block` 内。
```

## 修复原则

如果发现错位：

- 不要用更复杂的跨行表格修补。
- 改成更简单的组行结构。
- 缩小列数，而不是让中文长文本硬塞进窄列。
- 可以把“镜头编号/景别/摄影机/动作”合并为一个“镜头分解”单元格。
- 提示词列保持独立，宽度足够，并允许换行。

## 交付前状态行

最终回复或 HTML 内部备注可以包含：

```text
网页布局检查：已检查，无 rowspan/colspan，长提示词可换行，表格可横向滚动。
```
